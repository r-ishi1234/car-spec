## 構成

- **フロント**: Vite + React（開発: `npm run dev` → http://localhost:5173）
- **API**: Express + sql.js（SQLite）。開発時 API は http://localhost:3001（Vite が `/api` をプロキシ）
- **データ**: `data/specs.db`、`server/seed-vehicles.json`
- **本番ビルド**: `npm run build`（`dist-server` + `dist`）、起動: `npm start`
- **コンテナ**: ルートに `Dockerfile`（Render / GCP Cloud Run 向け）

- `src/api/vehicles.ts`: 本番（`PROD`）では API を常に相対 URL（`localhost` が焼き付かないように）

## GCP（Cloud Run）

- API 有効化: **Cloud Run Admin API**、**Artifact Registry API**、**Cloud Build API**
- Artifact Registry に Docker リポジトリ作成（例: リージョン `asia-northeast1`）
- Cloud Shell 等で:

  ```bash
  gcloud builds submit --tag asia-northeast1-docker.pkg.dev/<PROJECT_ID>/<REPO>/car-spec:latest .
  gcloud run deploy car-spec \
    --image asia-northeast1-docker.pkg.dev/<PROJECT_ID>/<REPO>/car-spec:latest \
    --region asia-northeast1 \
    --allow-unauthenticated \
    --port 8080
  ```

- **Forbidden**: 多くは「認証が必要」のまま。**未認証の呼び出しを許可**または IAM で **Cloud Run 起動元**。組織ポリシーで `allUsers` が禁止されることがある。
- **IP 制限**: Cloud Run 単体では不可。**外部 HTTPS LB + Cloud Armor**（許可 CIDR）が定石。別案は Cloudflare 等の前段。

## ローカルでポート競合（EADDRINUSE 3001）

別プロセスが 3001 を使用中。タスクマネージャで Node を終了するか、`npm run dev` だけ使う。
